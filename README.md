# HTML5 Game Player 🎮

**HTML5 Game Player** — это универсальный движок-шаблон и набор утилит для превращения ваших браузерных HTML5-игр в полноценные десктопные приложения (exe/elf/app) без использования тяжелого Electron. 
Проект написан на Go и использует нативный системный WebView для максимальной производительности и минимального веса.

---

## 🛠 Установка зависимостей (Если у вас нет Go)

Для использования инструментов из этого репозитория вам необходимо установить язык программирования **Go (Golang)**.

### Windows
1. Перейдите на официальный сайт [go.dev/dl](https://go.dev/dl/).
2. Скачайте установщик для Windows (`.msi`).
3. Установите программу, нажимая "Далее".
4. Откройте консоль PowerShell или CMD и введите `go version`, чтобы убедиться, что всё работает.

### macOS
1. Скачайте `.pkg` установщик с [go.dev/dl](https://go.dev/dl/).
2. Или установите через Homebrew в терминале: `brew install go`

### Linux (Ubuntu/Debian)
1. Выполните в терминале команду установки Go:
   ```bash
   sudo apt update
   sudo apt install golang-go
   ```
2. **ВАЖНО:** Для компиляции оконного приложения на Linux вам также понадобятся инструменты языка C и библиотеки GTK. Установите их командой:
   ```bash
   sudo apt-get install build-essential libwebkit2gtk-4.0-dev
   ```

## 🚀 1. Использование Универсального Launcher'а (Самый простой путь)

В проекте есть готовая программа `HTML5Launcher`, которая превратит любую папку с игрой в десктопное приложение без лишней компиляции.

1. **Скомпилируйте лаунчер один раз:**
   Откройте терминал в папке этого репозитория и выполните сборку для вашей ОС:

   **Для Windows:**
   ```powershell
   go build -o builds/HTML5Launcher.exe ./cmd/launcher
   ```

   **Для Linux / macOS:**
   ```bash
   go build -o builds/HTML5Launcher ./cmd/launcher
   ```

2. **Как пользоваться:**
   - Скопируйте полученный исполняемый файл (`HTML5Launcher.exe` на Windows или `HTML5Launcher` на Linux/macOS) в папку с вашей игрой (туда, где лежит файл `index.html`).
   - Запустите его (двойным кликом или через терминал: `./HTML5Launcher`).
   - Рядом с ним автоматически появится файл настроек `html5player_config.json`.
3. **Настройка (html5player_config.json):**
   Откройте этот файл в блокноте и вы сможете на лету менять параметры:
   - `Title`: Название окна вашей игры.
   - `Width` и `Height`: Разрешение окна.
   - `MuteAudio`: `true` отключит весь звук в игре.
   - `DevTools`: `true` включит панель разработчика браузера (F12).

## 🏗 2. Утилита для полной сборки и защиты (cmd/build)

Если вы хотите встроить игру внутрь одного `.exe` файла или зашифровать ресурсы, используйте встроенную утилиту сборки.

### Режим `embed` (Один файл)
Все ресурсы зашиваются внутрь исполняемого файла.
1. Положите файлы вашей игры (index.html и др.) в папку `game_data`.
2. Запустите сборку:
   ```powershell
   go run ./cmd/build build -mode embed -out my_game.exe
   ```

### Режим `xor` (Зашифрованный архив)
Ресурсы пакуются в зашифрованный (XOR) архив. Пользователи не смогут украсть ваши картинки или код.
1. Упакуйте ресурсы из папки `game_data` и задайте секретный пароль:
   ```powershell
   go run ./cmd/build pack -in game_data -out game_res.pak -key MySecretPassword
   ```
2. Соберите саму игру (с указанием того же пароля):
   ```powershell
   go run ./cmd/build build -mode xor -out my_game.exe -key MySecretPassword
   ```
*(Для распаковки своего архива используйте: `go run ./cmd/build unpack -in game_res.pak -out open.zip -key MySecretPassword`)*

### Режим `dir` (Внешняя папка)
Игра читает файлы напрямую из папки рядом (удобно для модов).
```powershell
go run ./cmd/build build -mode dir -out my_game.exe
```

## 📦 Пакетная сборка (Несколько игр сразу)

Если у вас есть несколько готовых игр в разных папках, вы можете скомпилировать их все одной командой:
```powershell
go run ./cmd/build batch -mode embed -games "C:\Games\mario,C:\Games\tetris"
```
Утилита сама извлечет имена папок (`mario`, `tetris`), бережно скопирует их, скомпилирует `.exe` файлы с соответствующими именами и сложит их в папку `builds/`. Работает со всеми тремя режимами.

## 🖼 Настройка иконки и мета-данных (Только для Windows)

При любой сборке через `cmd/build`, утилита автоматически "вшивает" в игру красивую иконку и данные разработчика.
В корне проекта есть папка `winres`:
1. Замените `icon.png` и `icon16.png` в этой папке на вашу собственную иконку.
2. Откройте файл `winres.json` и заполните данные: `CompanyName`, `FileDescription`, `LegalCopyright`, версию приложения и т.д.
3. Просто запустите любую команду `go run ./cmd/build ...`. Иконка автоматически появится у скомпилированного `.exe` файла!

## 🔌 JavaScript API (Связь между игрой и ОС)

Вы можете вызывать нативные функции языка Go прямо из JavaScript кода вашей игры!
Откройте файл `internal/api/api.go` и посмотрите пример функции `Ping`.
В вашем `index.html` (в JS) вызовите её:
```javascript
window.api.Ping("Привет из JS!").then(result => {
    console.log(result); // Получит ответ от Go
});
```
Сюда можно добавлять любые функции: сохранение сохранений на диск, работу с локальной сетью или вызов системных диалогов (выбор файла).
