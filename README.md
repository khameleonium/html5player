# HTML5 Game Player 🎮🚀

Короче, пацаны-ребята и клёвые девчата, это **HTML5 Game Player** — ультимативная пушка для того, чтобы превратить твою HTML5-игру в полноценное десктопное приложение под Windows, macOS или Linux. 
Изначально эта штука задумывалась чисто под HTML5-игры, но, будем честны, она легко переварит и упакует вообще любое веб-приложение. Так что если ты клепаешь на JS, HTML и CSS — ты по адресу.

> **Важно:** Во всех примерах ниже я пишу `.exe` чисто для удобства и потому что Windows пока рулит балом на десктопах. Если ты сидишь на Linux или macOS — смело убирай `.exe`, там бинарникам расширения не нужны.

---

## 🛠 Зачем это нужно?
Вместо того чтобы тянуть за собой жирный Electron, который жрет оперативу как не в себя, мы юзаем легкий и шустрый нативный движок (WebView2 на винде и WebKit на остальных). Твой код крутится на локальном сервере, написанном на мощном Go, а юзер видит только твою игру.

## 🚀 Фичи, за которые не стыдно:
- **Никаких костылей**: кроссплатформенная поддержка (Windows, macOS, Linux).
- **Своя атмосфера**: хочешь — будет окно, хочешь — полный экран (по `Ctrl+Alt+Enter`), хочешь — безрамочный режим.
- **Всё под контролем**: можно вырубить контекстное меню (чтобы юзеры не лазили куда не надо), звук и средства разработчика.
- **Защита от умников**: есть крутые режимы сборки с XOR-шифрованием, чтобы твои спрайты и код никто не стянул.
- **Двусторонняя связь**: JavaScript легко общается с Go через встроенный мост.

## 📦 Как установить эту красоту?

Если у тебя еще нет Go, самое время стать серьезным челиком:
1. Залетай на [официальный сайт Go](https://go.dev/dl/).
2. Скачай установщик под свою ОС.
3. Установи (там всё просто, далее-далее-готово).
4. Открой терминал и проверь, что всё работает: `go version`.

Теперь клонируй репозиторий, если еще этого не сделал:
```bash
git clone https://github.com/khameleonium/html5player.git
cd html5player
```

## 🎮 Шаг 1. Настройка игры

Твои исходники (HTML, JS, картинки, звуки) должны лежать в папке `game_data`. Главный файл обязательно должен называться `index.html`.

Рядом с бинарником создастся файл конфига `html5player_config.json`. Там можно подкрутить всё под себя:
- `Title`: Название твоей игры (будет в шапке окна).
- `Width` / `Height`: Размеры окна (если оно не на весь экран).
- `Fullscreen`: `true` — игра стартует на весь экран. Выйти/зайти обратно можно по `Ctrl+Alt+Enter`.
- `Frameless`: `true` — сносит рамку окна напрочь (стильно, модно, молодежно).
- `MuteAudio`: Вырубить звук (`true`/`false`).
- `DevTools`: Разрешить консоль разработчика (`true`/`false`).
- `ContextMenu`: `true` — разрешить правый клик (лучше оставь `false`, если не хочешь, чтобы юзеры видели кнопку "Обновить").

> **Совет от бывалых:**
> Конфиг читается лаунчером при запуске. Но когда ты собираешь бинарник через наш упаковщик `cmd/build`, все эти настройки **намертво вшиваются** прямо в файл! Так что файл конфига рядом с финальной игрой таскать не придется.

## 🔫 Шаг 2. Как собирать бинарник (cmd/build)

Тут у нас есть несколько режимов сборки. Выбирай свой стиль:

### Режим `embed` (Всё в одном)
Твоя игра просто вшивается внутрь бинарника. Никаких лишних файлов, просто один запускаемый файл.
```powershell
go run ./cmd/build build -mode embed -out my_game.exe
```

### Режим `embed-xor` (Броня в один файл)
Самый параноидальный и крутой режим. Мы пакуем ресурсы в зашифрованный архив, а потом этот архив вшиваем прямо в бинарник. При запуске всё расшифровывается прямо в оперативе — на диск ни байта не падает!
```powershell
go run ./cmd/build build -mode embed-xor -out my_game.exe -key MySecretPassword
```

### Режим `xor` (Внешний сейф)
Ресурсы шифруются паролем в отдельный файл `.pak`. Никто не украдет твои ассеты.
1. Пакуем `game_data` в архив с паролем:
   ```powershell
   go run ./cmd/build pack -in game_data -out game_res.pak -key MySecretPassword
   ```
2. Собираем лаунчер, который умеет его читать:
   ```powershell
   go run ./cmd/build build -mode xor -out my_game.exe -key MySecretPassword
   ```

### Режим `dir` (Классика)
Лаунчер просто читает ресурсы из папки рядом (по умолчанию `game_data`). Удобно для дебага и тестов.
```powershell
go run ./cmd/build build -mode dir -out my_game.exe
```

## 🏭 Пакетная сборка (для оптовиков)

Если у тебя много игр и ты хочешь собрать их все разом — не вопрос. Закидывай пути через запятую:
```powershell
go run ./cmd/build batch -mode embed -games "C:\Games\mario,C:\Games\tetris"
```
Скрипт пережует всё это и выплюнет готовые сборки в папку `builds/`. Красота!

## 🖼 Иконка и красивости (Только Windows)

Чтобы твой `.exe` выглядел солидно, закинь `icon.png` и `icon16.png` в папку `winres`. 
Там же лежит `winres.json`, где можно вписать имя компании, копирайты и версию.
Сборщик сам всё подхватит и приклеит к твоему бинарнику.

## 🤝 JavaScript API (Связь с базой)

Твой JS-код может общаться с ядром на Go! В файле `internal/api/api.go` лежит пример функции `Ping`.
В `index.html` дёргаешь её так:
```javascript
window.api.Ping("Здарова от JS!").then(result => {
    console.log(result); // Получишь ответочку от Go
});
```
Добавляй свои функции, обращайся к файловой системе, базам данных или чему угодно. Ограничений нет.
